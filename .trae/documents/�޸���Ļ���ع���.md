## 问题分析

通过调试脚本，我发现了无法下载字幕的根本原因：

1. 当解析字幕详情页时，代码正确找到了标题为"字幕文件下载"的h3元素
2. 但是使用`find_next()`方法获取下一个元素时，返回的是一个"返回首页"的链接，而不是包含下载链接的容器
3. 实际包含下载链接的容器是h3的兄弟元素，class为`list`的div，这个div包含了41个下载链接

## 修复方案

需要修改`src/samfunny/client.py`文件中的`parse_detail`方法，将第146行的代码从：
```python
container = section.find_next() if section else soup
```

修改为：
```python
container = section.find_next_sibling(class_="list") if section else soup
```

或者更健壮的实现，遍历所有兄弟元素直到找到包含下载链接的容器，以确保在不同页面结构下都能正确工作。

## 预期效果

修复后，字幕下载功能应该能够正确识别并解析下载链接，从而成功下载字幕文件。
