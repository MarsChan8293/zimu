# 将zimu项目打包为全局可执行安装包

## 目标
将当前的zimu软件打包为一个Python安装包，使得用户可以通过`pip install`命令安装，然后在任意目录下执行`zimu`命令来下载字幕，而不需要指定执行文件路径。

## 实现步骤

1. **创建setup.py配置文件**
   - 在项目根目录创建setup.py文件
   - 配置包的基本信息（名称、版本、作者等）
   - 定义依赖项（从requirements.txt导入）
   - 设置入口点，使`zimu`命令指向cli.py的main函数

2. **添加必要的__init__.py文件**
   - 在src目录下创建__init__.py文件，使其成为Python包
   - 确保samfunny子包的__init__.py文件配置正确

3. **调整项目结构**
   - 确保zimu.py的功能可以通过包的入口点访问
   - 移除或调整zimu.py中可能影响包安装的代码

4. **测试安装和执行**
   - 使用`pip install -e .`进行本地开发安装
   - 在不同目录下测试`zimu`命令是否正常工作
   - 确保所有功能与原项目一致

## 预期结果
- 用户可以通过`pip install .`或`pip install -e .`安装zimu
- 安装后可以在任意目录下直接执行`zimu`命令
- 命令功能与原项目完全一致
- 支持所有原有的命令行参数

## 技术要点
- 使用setuptools进行包管理
- 设置console_scripts入口点实现全局命令
- 保持项目原有功能不变
- 确保跨平台兼容性

通过以上步骤，我们可以将当前的zimu项目转换为一个标准的Python安装包，实现全局执行的功能。
